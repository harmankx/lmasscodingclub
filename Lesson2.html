The Basics

Syntax

Java, like many other coding languages, is case sensitive. This means that it matters if you use capital or lowercase letters. Java needs everything to stay consistent, if not the program will not recognize what you have written. For example, if you declare a variable “double pi = 3.14;” the variable “pi” must stay lowercase or it will not be recognized. We will touch more on this later what the meaning of double is and the different datatypes.

Classes

A class is nothing but a blueprint or a template for creating things which define its properties and behaviors. You would write your code in the main() method which is in a class. Declaring a class would look something like this:

public class Class_Name {
public static void main(String args[]) {

//your code here

}
}

You will notice in my previous example there is a piece of code that says “public static void main(String[] args)”. Every java program has to have this, otherwise your code will not work. This basically acts as a place for you to write your code. You can see there are different qualifiers written in front of the main() method. The qualifier “public” means this accessible across methods. “static” means it belongs to the entire class. “void” means it will not return a value. And as for main(), it is an entry point for your code. You MUST name it “main” or your code will not run.

Whitespace

Your code must be easy to read, which is why I want to talk about whitespace. When I say “whitespace”, essentially what I am talking about is how your code is written. In order for your code to be coherent to yourself and others, it is important to leave space between code. For example if your code is written like this:

public class Whitespace {
public static void main(String args[]) {
int num1 = 5;
int num2 = 4;
int num3 = num1 + num2;
System.out.println(num1);
System.out.println(num2);
System.out.println(num3);
}
}

This is what we call bad code. It’s bad because it’s is not very clear to the user. This is why it is important to use whitespace; in order to make it look neat! For example:

public class Whitespace {
public static void main(String args[]) {

int num1 = 5;
int num2 = 4;
int num3 = num1 + num2;

System.out.println(num1);
System.out.println(num2);
System.out.println(num3);

}
}

It’s easier to see because the int declarations are separated from the outputs. While this is all well and good, there is such a thing as too much white space. You don’t want your code to be too spread out as it will look unorganized. For example:

public class Whitespace {
public static void main(String args[]) {

int num1 = 5;

int num2 = 4;

int num3 = num1 + num2;

System.out.println(num1);

System.out.println(num2);

System.out.println(num3);

}
}

This is another example of bad code as the lines are too spaced out!

A Basic Output

From the last example you’ve seen me use a command that looks like this:

System.out.println()

This is used to output something to the user and end the line whether it is a variable or just a couple words. To output a word you would write your word in quotation marks within the parentheses. For example:

public class MyFirstWords {
public static void main(String args[]) {

System.out.println("Hello World!");

}
}

This line of code will output Hello World!

There are other commands to output, such as the System.out.print() command, where this will output something and will not end the line. For example if you made this program:

public class MySecondWords {
public static void main(String args[]) {

System.out.print("Wow")
System.out.print("so cool");

}
}

It will output “Wow so cool!” to the user, instead of putting them on different lines. If you used the System.out.println() command, your code would have outputted "Wow" on one line, and "so cool!" on then next line!
